# User Story: Configure Opencode Config via install.sh
- ID: US-0001
- Status: plan
- Feature: FEAT-0001
- As a: repository user
- I want: a POSIX install.sh that configures ~/.config/opencode with this repo’s agents/skills/commands
- So that: I can set up or refresh my opencode configuration reliably
- Acceptance criteria:
  - Given the repository contains agents, skills, and commands
  - When I run install.sh using POSIX sh
  - Then ~/.config/opencode is created if missing and updated to match the repository
  - Given I have already run install.sh once
  - When I run it again
  - Then it completes successfully without changing the desired end state
  - Given I do not have elevated privileges
  - When install.sh runs
  - Then it completes without invoking sudo
