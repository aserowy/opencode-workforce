# Task: Specify output formatting for readability

- ID: TASK-0009
- Status: plan

## Description
Define the formatting conventions for progress output and the end-of-run summary to ensure readability in plain-text terminals using POSIX sh.

## Scope
- Specify section headings and spacing for progress and summary blocks.
- Define consistent prefixes (e.g., "[INFO]", "-") for progress lines.
- Outline summary layout (header, counts block, optional notes).
- Ensure formatting remains plain-text friendly without non-POSIX features.

## Acceptance Criteria
- Formatting rules are documented with example snippets.
- Progress and summary formats are consistent with each other.
- Output remains readable in plain text terminals.
